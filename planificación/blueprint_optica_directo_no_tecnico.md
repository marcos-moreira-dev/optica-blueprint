# Blueprint directo para sistema integral de óptica

## Idea rectora

El sistema debe sentirse como una herramienta de trabajo seria para una óptica real. No debe parecer una aplicación genérica ni una pantalla recargada de tecnología. Debe verse como una mezcla equilibrada entre atención visual, operación comercial, control de pedidos y seguimiento al cliente.

La óptica no es solo una tienda ni solo un consultorio. Es un negocio híbrido. Por eso la interfaz debe transmitir tres cosas al mismo tiempo:
- orden clínico
- control comercial
- seguimiento operativo

## Carácter visual general

La apariencia debe ser limpia, sobria y profesional. La sensación debe ser más cercana a un negocio óptico organizado que a un sistema hospitalario pesado o a una tienda ruidosa.

Debe transmitir:
- limpieza
- precisión
- confianza
- calma
- claridad
- control diario

No debe transmitir:
- exceso de decoración
- saturación de colores
- ambiente infantil
- estilo de dashboard de marketing
- sensación de software improvisado

## Lenguaje visual recomendado

La óptica vende salud visual, precisión y estética. Por eso el sistema debe usar un lenguaje visual contenido, elegante y funcional.

### Decisiones visuales base
- fondos claros
- superficies limpias
- contraste suficiente para leer muchas horas
- acentos fríos y sobrios
- esquinas discretas, casi rectas
- bordes suaves pero no juguetones
- muy poco adorno gratuito

### Sensación artística buscada
- clínica ligera, no hospitalaria
- retail premium, no lujoso exagerado
- administración seria, no burocrática

## Paleta más probable para una óptica

Tomando en cuenta el tipo de negocio, lo más probable y conveniente es una paleta fría, limpia y estable.

### Colores base
- blanco roto o gris muy claro para fondo general
- blanco limpio para superficies principales
- azul grisáceo o azul acero suave para estructura
- azul petróleo moderado para énfasis principal

### Colores de estado
- verde moderado para listo, confirmado, entregado
- ámbar sobrio para pendiente, por revisar, por cobrar
- rojo controlado para cancelado, vencido, error, retrasado
- azul secundario para información neutral

### Regla importante
El color principal del sistema no debe pelear con los colores de estado. Los estados deben servir para operar, no para decorar.

## Navegación principal

El sistema debe tener una navegación lateral fija y muy clara. La óptica necesita cambiar rápido entre atención, ventas, inventario, órdenes y reportes. La navegación debe ayudar a trabajar, no a explorar.

### Estructura recomendada del menú principal
- Inicio
- Agenda
- Clientes
- Recetas
- Venta óptica
- Órdenes de laboratorio
- Inventario
- Caja
- Entregas
- Seguimiento
- Reportes
- Configuración

### Módulos extendidos
- Seguros / Cobertura
- Proveedores
- Compras
- Usuarios y roles
- Taller / Reparaciones
- Notificaciones
- Sucursales

### Cómo debe verse el módulo activo
- claramente marcado
- sin exageración visual
- con una diferencia obvia respecto al resto
- fácil de reconocer al primer vistazo

## Encabezado común de cada módulo

Cada módulo debe comenzar con una franja clara y estable que permita al usuario ubicarse rápido.

### Debe incluir
- nombre del módulo
- una línea breve que explique su propósito
- acciones principales del módulo
- referencia de sucursal si aplica

### Debe evitar
- adornos innecesarios
- frases genéricas sin valor
- encabezados gigantes

## Búsqueda y filtros

En una óptica real, muchas tareas empiezan buscando a alguien, una orden, una montura, un pendiente o una receta. Por eso los filtros no son un extra: son parte central del trabajo.

### Qué debe ser común
- búsqueda visible
- filtros comprensibles
- limpieza rápida de filtros
- lectura clara de qué criterio está activo

### Lo más común en la óptica
- cliente
- fecha
- estado
- sucursal
- profesional
- tipo de orden
- marca
- proveedor
- saldo pendiente

## Criterio visual para tablas

Gran parte del trabajo diario va a pasar por tablas. Por eso no deben verse improvisadas.

### Cómo deben sentirse
- claras
- respirables
- ordenadas
- con jerarquía visible
- cómodas para jornadas largas

### Lo que debe destacar en una tabla óptica
- cliente
- orden
- estado
- fecha promesa
- saldo
- sucursal
- marca o producto según el caso

### Acciones dentro de tabla
Las acciones más delicadas no deben estar perdidas ni mezcladas. Editar, cobrar, entregar, reprogramar o ver detalle deben estar siempre en lugares predecibles.

## Criterio visual para fichas y detalle lateral

La óptica trabaja mucho con contexto. No basta ver una fila; hace falta ver historia, receta, compra, estado y observaciones. Por eso conviene que muchos módulos tengan una zona de detalle visible.

### Qué debe mostrar el detalle
- resumen del cliente
- receta relacionada
- estado del trabajo
- observaciones relevantes
- saldo o situación de pago
- historial reciente

### Cómo debe sentirse
- útil
- rápido de leer
- sin ruido visual
- más informativo que decorativo

## Criterio visual para formularios

Los formularios deben verse serios y precisos. En óptica, un dato mal puesto puede afectar receta, pedido, cobro o entrega.

### Deben transmitir
- orden
- precisión
- claridad
- flujo natural

### Regla general
Los campos deben agruparse por significado real, no por relleno visual.

### Agrupaciones típicas
- datos del cliente
- datos de contacto
- receta
- producto o montura
- lentes y tratamientos
- pago
- entrega
- observaciones

## Criterio visual para procesos por etapas

Hay procesos en óptica que no conviene resolver en una sola pantalla caótica. El caso más claro es la venta óptica completa.

### Casos que sí justifican etapas
- crear orden óptica
- registrar reparación
- trabajar convenios complejos

### Cómo debe sentirse un flujo por etapas
- guiado
- claro
- progresivo
- difícil de romper por error

### Qué debe dejar claro
- en qué paso está el usuario
- qué falta
- qué ya quedó completado
- qué no se puede avanzar todavía

## Gramática visual de estados

Una óptica depende mucho de estados. Un sistema sin estados claros se vuelve caótico muy rápido.

### Estados más probables
- confirmado
- pendiente
- cancelado
- en proceso
- listo
- entregado
- por cobrar
- vencido
- bajo stock
- agotado

### Regla visual
Cada estado debe verse claro en texto y no depender solo del color.

## Multi-sucursal

Como una óptica puede tener dos o más sedes, el sistema debe estar preparado visualmente para eso desde el principio.

### Qué debe quedar claro
- en qué sucursal se está trabajando
- a qué sucursal pertenece una orden
- dónde está el inventario
- dónde se entrega el trabajo
- qué ventas o citas corresponden a cada sede

La referencia de sucursal debe estar presente, pero sin ensuciar la pantalla.

## Tono del texto del sistema

El texto visible debe sonar profesional, corto y útil.

### Debe ser
- claro
- concreto
- amable sin exceso
- orientado a acción

### No debe ser
- técnico innecesario
- rebuscado
- ambiguo
- demasiado informal

## Tooltips y ayudas

Las ayudas deben existir solo donde realmente ahorran dudas.

### Lugares donde sí convienen
- acciones delicadas
- estados poco obvios
- abreviaturas ópticas si se usan
- operaciones con impacto económico
- cambios de estado

### Lugares donde no convienen tanto
- botones obvios
- texto ya completamente claro
- elementos repetidos sin valor adicional

## Qué debe verse en el inicio del sistema

La pantalla inicial no debe ser solo decorativa. Debe ser un centro de control diario.

### Lo más probable y útil
- citas del día
- ventas del día
- órdenes pendientes
- trabajos listos para entregar
- productos con stock crítico
- avisos importantes

## Qué debe sentirse en los módulos principales

### Agenda
Orden temporal y claridad de atención.

### Clientes
Ficha continua, no solo lista de nombres.

### Recetas
Precisión técnica y estructura clara.

### Venta óptica
Flujo guiado y armado correcto del trabajo.

### Órdenes de laboratorio
Seguimiento logístico visible.

### Inventario
Control real de variantes, no conteo superficial.

### Caja
Rapidez, orden y cierre claro.

### Entregas
Validación final y control de pendientes.

### Seguimiento
Relación activa con el cliente.

### Reportes
Lectura gerencial sobria.

### Configuración
Orden interno, no protagonismo visual.

## Decisiones estéticas ya resueltas para este proyecto

Para mantener consistencia, se asumen estas decisiones como base:
- no abusar de tarjetas redondeadas
- preferir superficies limpias y rectangulares
- usar redondez mínima o muy discreta
- evitar sombras exageradas
- evitar gradientes llamativos
- evitar aspecto de app móvil juvenil
- priorizar orden visual por encima del adorno

## Qué falta antes de bajar a cada módulo

Ya no queda nada crítico por definir a nivel base. Para efectos de este proyecto conceptual, se asumen cerradas las siguientes convenciones globales.

## Convenciones globales ya decididas

### 1. Nombre exacto de cada módulo en el menú

#### Núcleo principal

#### Extendidos

- Sucursales

### 2. Color principal oficial del sistema

Se fija como color principal un **azul petróleo sobrio**, porque encaja bien con una óptica seria, transmite limpieza y profesionalismo, y no compite demasiado con los estados operativos.

Como apoyo visual, se asumen estos criterios:
- fondo general muy claro
- superficies blancas o casi blancas
- azul petróleo como color de identidad
- azul grisáceo como color de apoyo estructural
- grises suaves para divisiones y bordes

### 3. Estilo oficial del módulo activo en la navegación

El módulo activo en el menú lateral debe verse:
- con fondo ligeramente más oscuro o más denso que el resto
- con una línea vertical o acento visual discreto al costado
- con texto más firme que los otros módulos
- con ícono y texto claramente destacados, sin exageración

No se usará un estilo chillón ni una selección demasiado agresiva. La idea es que el usuario note rápido dónde está, pero sin ensuciar la interfaz.

### 4. Lista oficial de estados del sistema

Se toma como base esta gramática de estados, por ser la más útil y probable para una óptica:
- Activo
- Inactivo
- Pendiente
- Confirmado
- Cancelado
- En proceso
- Listo
- Entregado
- Por cobrar
- Pagado
- Vencido
- Bajo stock
- Agotado
- Requiere revisión

Cada estado debe expresarse con texto claro y apoyo visual consistente. Nunca debe depender solo del color.

### 5. Forma oficial de los botones principales, secundarios y peligrosos

#### Botón principal
- color sólido con el azul principal del sistema
- bordes discretos
- esquinas ligeramente suavizadas, pero no redondeadas de forma llamativa
- alto cómodo y lectura clara
- reservado para la acción más importante de la pantalla

#### Botón secundario
- estilo más neutro
- fondo claro o muy suave
- borde visible pero sobrio
- usado para acciones de apoyo, consulta o navegación operativa

#### Botón peligroso
- color de advertencia sobrio, no escandaloso
- debe diferenciarse de inmediato de las acciones normales
- usado solo para acciones realmente delicadas, como anular, eliminar o cerrar procesos sensibles

#### Regla general
No se abusará de botones gigantes, bordes inflados ni estilos decorativos. Los botones deben verse administrativos, no promocionales.

### 6. Tono oficial del microcopy del sistema

El texto del sistema se fija con este tono:
- directo
- profesional
- corto
- útil
- amable, pero sin exceso de cercanía

#### Ejemplos del estilo buscado
- “Nueva venta óptica”
- “Registrar entrega”
- “Filtrar por sucursal”
- “No hay órdenes pendientes”
- “Complete la receta antes de continuar”
- “Confirme la entrega del trabajo”

#### Lo que se evita
- frases largas e innecesarias
- tecnicismos de programación
- mensajes ambiguos
- tono demasiado informal
- lenguaje genérico que podría pertenecer a cualquier otro negocio

### 7. Estilo oficial de tooltips

Los tooltips se usarán solo donde aporten valor real.

#### Tono del tooltip
- breve
- instructivo
- claro
- sin redundancia

#### Casos donde sí se usarán
- acciones sensibles
- estados que puedan generar duda
- atajos operativos
- campos con interpretación técnica o ambigua

#### Casos donde no se usarán
- botones cuyo texto ya sea totalmente claro
- acciones repetitivas sin ambigüedad
- elementos decorativos

### 8. Estilo oficial de bordes, paneles y superficies

Para este proyecto se asume como regla visual definitiva:
- superficies limpias
- bordes discretos
- esquinas casi rectas o con suavidad mínima
- nada de paneles excesivamente redondeados
- nada de tarjetas infladas estilo app móvil juvenil
- muy poca sombra
- separación clara entre bloques por orden y no por exceso de efecto visual

### 9. Regla de consistencia transversal

Todos los módulos deberán mantener:
- mismo tono visual
- misma lógica de encabezado
- misma seriedad en los textos
- misma familia de estados
- misma lógica de botones
- misma sobriedad general

El objetivo es que, aunque cada módulo tenga su propia función, todo el sistema se sienta como una sola familia visual y operativa.

## Cierre

Este blueprint no presenta una aplicación genérica. Presenta la base visual y operativa de un sistema hecho para una óptica real: una empresa que atiende clientes, gestiona recetas, arma trabajos, controla inventario, da seguimiento a pedidos, cobra, entrega y, en muchos casos, opera en más de una sede.

La meta no es llenar pantallas. La meta es que el sistema se sienta propio del rubro óptico, útil en el trabajo diario y coherente en toda la experiencia.
