# Guía visual transversal de iconografía, tipografía y recursos gráficos para sistema de óptica

## Propósito

Este documento fija una línea visual transversal para todo el sistema de óptica. Su función es ayudar a que el producto se vea más serio, más coherente y más propagandístico, sin depender de imágenes complejas ni de recursos difíciles de producir.

La idea es apoyar la interfaz principalmente en:
- íconos
- tipografía
- color
- jerarquía visual
- bloques limpios

No se busca hacer arte complejo. Se busca una apariencia sólida, consistente y creíble para una óptica moderna.

---

## 1. Regla general de recursos visuales

### Decisión oficial
La interfaz debe apoyarse principalmente en **íconos** y **tipografía**, no en fotografías ni ilustraciones complejas.

### Motivos
- es más fácil de implementar
- es más fácil de mantener
- se ve más coherente entre módulos
- encaja mejor con un sistema administrativo desktop
- sirve muy bien para maquetas propagandísticas

### Regla práctica
Si existe duda entre usar una imagen o un ícono, debe preferirse el ícono.

---

## 2. Uso de imágenes

### Regla base
Las imágenes no serán el recurso principal del sistema.

### Cuándo sí podrían usarse
- logo del negocio
- un recurso muy puntual en catálogo, si en el futuro se quisiera
- una ayuda visual muy específica, si realmente aporta

### Cuándo no conviene usarlas
- en módulos administrativos normales
- en tablas
- en paneles de detalle que ya funcionan con texto e íconos
- como relleno decorativo
- para representar personas o personajes

### Decisión de este proyecto
Para este sistema conceptual, se asume que la gran mayoría de pantallas puede funcionar correctamente **sin imágenes reales**.

---

## 3. Estilo general de iconografía

### Carácter del ícono
Los íconos deben ser:
- sobrios
- claros
- modernos pero discretos
- fáciles de reconocer
- consistentes entre sí

### Estilo recomendado
- lineal o semi-lineal
- con grosor consistente
- con geometría limpia
- sin exceso de detalle interior

### Estilo que se debe evitar
- caricaturesco
- demasiado decorativo
- demasiado infantil
- muy grueso o muy pesado
- mezcla de estilos incompatibles

### Sensación buscada
Los íconos deben sentirse como parte de una herramienta profesional, no como adornos.

---

## 4. Dónde sí usar íconos

Se recomienda usarlos en:
- menú lateral principal
- títulos de ciertas tarjetas o bloques, cuando realmente aporten
- accesos rápidos
- acciones de tablas, si ayuda a ahorrar espacio
- catálogos sin imagen real
- estados operativos puntuales
- submódulos internos cuando necesiten diferenciación rápida

## Dónde no abusar de íconos
No conviene saturar con íconos:
- cada label de formulario
- cada fila de tabla sin necesidad
- textos largos
- subtítulos descriptivos
- paneles ya suficientemente claros con texto

### Regla de oro
El ícono debe acelerar la lectura, no ensuciar la pantalla.

## 6. Uso de íconos en catálogos sin fotografías

### Regla oficial
Cuando un catálogo no utilice imágenes reales, cada categoría debe apoyarse en un ícono representativo.

### Ejemplos más probables
- montura: ícono de gafas o armazón
- lente: ícono de lente circular
- lente de contacto: ícono circular más simple
- accesorio: ícono neutro según categoría
- reparación: ícono de herramienta
- receta: ícono de ojo o ficha técnica

### Ventaja
Esto permite que el catálogo se vea ordenado y profesional sin depender de fotos, renders ni material gráfico costoso.

---

## 7. Color aplicado a íconos

### Regla base
Los íconos no deben competir con el contenido textual.

### Criterio recomendado
- color estructural del sistema para navegación principal
- gris medio para íconos secundarios
- color de estado solo cuando el ícono represente una alerta o situación operativa

### Uso sugerido
- azul petróleo para navegación principal
- gris azulado para apoyo visual
- verde moderado para listo, confirmado o entregado
- ámbar para pendiente o por revisar
- rojo sobrio para cancelado, vencido o error

### Lo que se debe evitar
- muchos colores distintos en una misma pantalla
- íconos decorativos de colores intensos
- saturar la interfaz con color innecesario

---

## 8. Regla sobre tamaño de íconos

### Tamaño pequeño
Para acciones por fila o contexto secundario.

### Tamaño medio
Para menú lateral, botones importantes y títulos de bloque.

### Tamaño grande
Solo para tarjetas KPI o accesos rápidos muy puntuales.

### Regla general
No hacer íconos gigantes. La jerarquía debe seguir dominada por la estructura y por el texto.

---

## 9. Tipografía general del sistema

### Objetivo tipográfico
La tipografía debe ayudar a que el sistema se vea:
- claro
- estable
- administrativo
- moderno
- fácil de leer durante horas

### Carácter de la tipografía
Debe sentirse más cerca de un software empresarial limpio que de una aplicación promocional o publicitaria.

### Criterios deseados
- sans serif
- muy legible en tamaños medianos y pequeños
- buena lectura en tablas
- buena lectura en formularios
- buena lectura en botones y estados

---

## 10. Jerarquía tipográfica

### Nivel 1: título de módulo
Ejemplos:
- Inicio
- Agenda
- Clientes
- Inventario

Debe ser el texto más fuerte del módulo.

### Nivel 2: subtítulo de módulo
Ejemplos:
- Resumen general de la operación diaria
- Control diario y semanal de citas

Debe orientar, no competir con el título.

### Nivel 3: título de bloque
Ejemplos:
- Próximas citas
- Pendientes y alertas
- Actividad reciente
- Accesos rápidos

Debe organizar visualmente el área de trabajo.

### Nivel 4: labels y títulos de campo
Ejemplos:
- Cliente
- Profesional
- Estado
- Fecha
- Tipo de atención

Debe ser claro, corto y consistente.

### Nivel 5: texto secundario
Uso para observaciones breves, contexto menor o ayuda suave.

Debe ser visible, pero no protagonista.

### Nivel especial: cifras clave
Ejemplos:
- ventas del día
- citas de hoy
- stock crítico
- órdenes pendientes

Estas cifras deben destacar con fuerza, pero sin parecer publicidad exagerada.

---

## 11. Reglas tipográficas prácticas

### Lo que sí debe ocurrir
- títulos cortos
- botones con verbos concretos
- labels simples
- subtítulos breves
- mensajes operativos directos

### Lo que se debe evitar
- frases largas dentro de la interfaz
- etiquetas ambiguas
- exceso de mayúsculas
- tipografía ornamental
- lenguaje técnico innecesario

---

## 12. Tipografía en tablas

Las tablas son parte central del sistema.

### Reglas
- nombres y referencias deben leerse con facilidad
- estados deben verse cortos y claros
- fechas y cifras deben quedar bien ordenadas
- las observaciones no deben ser demasiado largas

### Jerarquía dentro de la tabla
- nombre o referencia principal con más peso visual
- estado claro y rápido de identificar
- fecha y sucursal con buen orden
- observación breve y discreta

---

## 13. Tipografía en botones

### Regla general
Los botones deben sonar operativos, no comerciales.

### Ejemplos correctos
- Nueva cita
- Nuevo cliente
- Registrar entrega
- Ver inventario
- Editar receta
- Marcar confirmada
- Enviar recordatorio
- Abrir cliente

### Ejemplos a evitar
- Haga clic aquí
- Continuar operación inteligente
- Activar gestión avanzada

### Criterio
El texto del botón debe dejar claro qué hace sin adornos innecesarios.

---

## 14. Tipografía en estados

Los estados deben ser muy cortos y muy legibles.

### Ejemplos de estados correctos
- Confirmada
- Pendiente
- Cancelada
- Reprogramada
- Atendida
- En proceso
- Listo
- Entregado
- Por cobrar
- Agotado
- Bajo stock

### Regla
Nunca usar estados demasiado largos ni ambiguos.

---

## 15. Tipografía en tooltips

### Tono del tooltip
- breve
- claro
- útil
- instructivo

### Ejemplos correctos
- Ver la agenda detallada de esta fecha
- Registrar una nueva cita en la sucursal activa
- Marcar la cita como confirmada
- Consultar productos con existencias bajas o agotadas

### Lo que se debe evitar
- repetir exactamente lo mismo que ya dice el botón
- escribir tooltips demasiado largos
- usar lenguaje técnico o rebuscado

---

## 16. Uso de tipografía en catálogos sin imágenes

Cuando el sistema muestre productos sin fotos, la tipografía debe ayudar a que el catálogo siga viéndose serio.

### Cada ítem debe apoyarse en
- un ícono representativo
- nombre del producto
- referencia o código
- categoría o tipo
- estado de stock
- precio si aplica

### Regla
El nombre del producto debe mandar más que el ícono, pero el ícono debe ayudar a reconocer rápidamente la categoría.

---

## 17. Uso combinado de ícono + texto

Esta combinación debe ser la base del sistema.

### Fórmula recomendada
- ícono discreto
- texto claro
- color sobrio
- espaciado limpio

### Casos donde funciona muy bien
- botones de acceso rápido
- opciones del menú lateral
- categorías de catálogo
- bloques resumen
- acciones secundarias

### Casos donde debe usarse con moderación
- formularios extensos
- tablas muy densas
- paneles ya suficientemente cargados

---

## 18. Decisión oficial para este proyecto

Para que el sistema se vea propagandístico sin complicarse, se fija esta regla:

- la interfaz se apoyará en íconos antes que en imágenes
- la tipografía será una de las herramientas principales de jerarquía
- los catálogos podrán funcionar con ícono + nombre + código + estado
- no se diseñarán personajes ni ilustraciones propias
- no se dependerá de fotografías para que la interfaz se vea seria

---

## 19. Resumen ejecutivo

Este sistema debe verse sólido por su orden visual, no por adornos. La iconografía será funcional, coherente y moderada. La tipografía será clara, administrativa y jerárquica. Las imágenes reales no serán necesarias en la mayoría de módulos. Cuando haga falta representar productos, categorías, estados o acciones, se hará principalmente con íconos y texto.

La meta es simple: que la óptica se vea profesional, moderna y organizada sin obligarte a producir material gráfico complejo.
